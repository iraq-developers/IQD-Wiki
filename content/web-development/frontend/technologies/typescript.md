# TypeScript (TS)

## شنو يعني TypeScript؟

بكل بساطة: **TypeScript هي JavaScript بس لابسة قاط ورباط.**

هي لغة بنتها Microsoft لتطوير لغة JavaScript. أي كود JS يعتبر كود TS صحيح، بس العكس مو صحيح.
الفرق الرئيسي هو **Type Safety** (الأمان بالأنواع). يعني شنو؟

بالـ JavaScript، تكدر تسوي هيج شي خطأ بدون ما احد يكلك:

```javascript
let name = "Omar";
name = 123; // عادي جداً، بس ممكن يكسر الكود بعدين
```

بالـ TypeScript، راح يطلعلك خطأ مباشرة قبل ما تشغل الكود:

```typescript
let name: string = "Omar";
name = 123; // Error: Type 'number' is not assignable to type 'string'.
```

هذا الشي يمنع آلاف الأخطاء (Bugs) اللي ممكن تصير بالمشاريع الكبيرة.

---

## ليش نستخدم TypeScript؟

1.  **اكتشاف الأخطاء مبكراً:** بدال ما تنتظر المستخدم يكتشف الخطأ، الـ Editor راح يكلك عليه وأنت تكتب.
2.  **Auto-completion خرافي:** لأن اللغة تعرف نوع كل متغير، تكدر تقترح عليك الدوال والخصائص الصحيحة فقط.
3.  **سهولة الصيانة:** بالمشاريع الكبيرة، الـ TS تخلي الكود مفهوم وأسهل للتعديل لأنك تعرف كل دالة شنو تستلم وشنو ترجع.

## أساسيات الكتابة (Syntax Comparison)

الفرق بسيط جداً، بس نضيف ": Type" بعد المتغيرات والدوال.

### JavaScript

```javascript
function add(a, b) {
  return a + b;
}

add("1", 2); // النتيجة "12" (نص) ، وهذا خطأ كارثي بالحسابات!
```

### TypeScript

```typescript
function add(a: number, b: number): number {
  return a + b;
}

add("1", 2); // Error: Argument of type 'string' is not assignable to parameter of type 'number'.
```

## الخلاصة

تعلمك للـ TypeScript هو **نقلة نوعية** بمسيرتك المهنية. الشركات حالياً تعتبرها مهارة أساسية لأي مطور Frontend محترف.
ابدأ بـ JavaScript، بس خلي هدفك توصل لـ TypeScript بأسرع وقت.
